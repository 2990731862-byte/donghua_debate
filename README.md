# 东华大学辩手积分榜

> 东华大学辩论协会官方辩手积分系统 — 微信小程序

## 项目背景

为提升辩手比赛积极性、建立科学合理的激励机制，东华大学辩论协会设计并推出了**辩手积分榜制度**。本系统基于协会发布的《关于试行辩手积分榜制度的通知》，将积分规则数字化，实现比赛数据录入、自动积分计算和实时排行榜展示。

**核心目标：**
- 为辩手提供清晰可见的成长路径和荣誉体系
- 通过积分排名激发辩手参赛积极性与竞技热情
- 建立公平透明的数据化管理平台，替代传统人工统计

## 功能概览

### 辩手端（普通用户）

**排行榜**
- 实时查看辩手积分排名
- 支持按时间（月份）筛选
- 支持按赛事级别筛选：全部 / 校赛 / 市赛
- 展示积分、参赛场次、佳辩次数等数据

**辩手搜索**
- 按姓名搜索辩手
- 查看个人详细积分明细
- 查看历史参赛记录与各场比赛积分拆解

**关于页面**
- 完整积分规则说明
- 管理员入口（需邀请码验证）

### 管理端（管理员）

**数据概览仪表盘**
- 辩手总数、比赛总数统计
- 清空数据日期提醒
- 快捷操作入口

**比赛录入**
- 录入比赛名称、级别、日期、赛程、总轮次
- 添加参赛辩手及其参赛轮次、最佳辩手、前三名信息
- 支持同场比赛多轮次分批录入（自动合并）
- 自动防重复录入检测

**比赛管理**
- 查看、编辑、删除历史比赛记录
- 完整的比赛数据管理

**辩手管理**
- 添加、编辑、删除辩手信息
- 包含姓名、学号、学院等

**积分重算**
- 全量重新计算所有辩手积分
- 确保数据一致性

**数据清空**
- 清空比赛数据并重置所有积分（用于赛季切换）

**权限管理**
- 超级管理员可生成邀请码
- 邀请码授权管理员权限（7天有效期）
- 支持邀请码管理（查看、删除）

## 积分算法

### 公式

```
轮次积分 = 基础奖励 + 破纪录奖励
比赛总积分 = Σ 各轮次积分（向下取 0.5 的倍数）
```

### 基础奖励

| 项目 | 公式 |
|------|------|
| 轮次基础分 | 0.5 × W × 轮次号 |
| 最佳辩手 | 0.5 × W |
| 前三名（最终名次） | W |

### 赛事权重 W

| 赛事级别 | 权重 W |
|----------|--------|
| 院赛 | 1 |
| 校赛 | 2 |
| 市赛 | 4 |

### 破纪录奖励

当辩手在某级别赛事中**首次达到个人历史最高纪录**时，可获得额外奖励：

| 项目 | 公式 |
|------|------|
| 破纪录轮次分 | 0.5 × log₂(W) + 0.5 × ⌊0.5 × 轮次号⌋ |
| 破纪录佳辩分 | 0.5 × W |
| 破纪录前三分 | 0.5 × W |

> 破纪录按赛事级别独立计算，同级别赛事中首次达到即算破纪录。

### 参考积分表（基础奖励示例）

| 轮次 | 院赛 (W=1) | 校赛 (W=2) | 市赛 (W=4) |
|------|------------|------------|------------|
| 第1轮 | 0.5 | 1.0 | 2.0 |
| 第2轮 | 1.0 | 2.0 | 4.0 |
| 第3轮 | 1.5 | 3.0 | 6.0 |
| 第4轮 | 2.0 | 4.0 | 8.0 |

## 数据库设计

### 辩手集合 (debaters)
| 字段 | 说明 |
|------|------|
| name | 姓名 |
| studentId | 学号 |
| college | 学院 |
| totalScore | 总积分 |
| matchCount | 参赛场次 |
| personalRecords | 个人纪录（按级别：最高轮次、佳辩、前三） |
| rank | 排名 |

### 比赛集合 (matches)
| 字段 | 说明 |
|------|------|
| name | 比赛名称 |
| level | 赛事级别 |
| stage | 赛程阶段 |
| date | 比赛日期 |
| totalRounds | 总轮次 |
| weight | 赛事权重 |
| participants | 参赛者列表（含轮次、佳辩、积分明细） |

### 管理员集合 (admins)
| 字段 | 说明 |
|------|------|
| openid | 微信 openid |
| name | 管理员姓名 |
| role | 角色（super_admin / admin） |
| invitedBy | 邀请人 |
| expiresAt | 过期时间 |

## 云函数列表

| 函数名 | 说明 |
|--------|------|
| login | 用户登录，获取 openid 及管理员权限 |
| getLeaderboard | 获取排行榜数据（支持级别和月份筛选） |
| searchDebaters | 搜索辩手 |
| getDebaterDetail | 获取辩手详细信息 |
| submitMatch | 提交比赛记录并触发全局积分重算 |
| manageMatch | 管理比赛（编辑、删除） |
| manageDebater | 管理辩手（增删改） |
| recalculateScores | 全量重算积分 |
| checkAdmin | 验证管理员权限 |
| inviteAdmin | 生成/管理邀请码 |
| clearMatchData | 清空比赛数据并重置积分 |
| graduateCleanup | 毕业生数据处理 |


## 积分汇报机制

比赛结束后，队长或联系人须在 **3 个工作日内** 将比赛信息提交至辩协，信息包括：
- 比赛名称、时间
- 对手信息
- 比赛结果
- 上场队员及各轮次参赛情况
- 最佳辩手
- 获奖名次

## 部署说明

1. 使用微信开发者工具打开项目
2. 在 `miniprogram/app.js` 中配置云开发环境 ID
3. 上传云函数至云开发环境
4. 创建数据库集合：`debaters`、`matches`、`admins`、`invitations`
5. 设置数据库权限规则
6. 编译预览并提交审核
